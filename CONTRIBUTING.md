# Welcome

Welcome! We are glad you are interested in contributing to datadog-iac-scanner-github-action.

This guide will help you understand the requirements and guidelines to improve your contributor experience.

## Reporting bugs

If you think you've found a bug, please search the issue list to check if it's been reported before. If you found nothing, please open a GitHub issue.

## Pull requests

If you want to contribute a new feature, please open a GitHub issue before you start writing code. This will avoid going through the effort of writing the code and getting it rejected because we are already working on it or because it's outside the scope of our project.

When you send us a pull request, follow these tips to make the review easier and faster:

* Open your PR against the `main` branch.
* Write an explanatory title and message for your PR, including references to any other GitHub issues or PRs.
* Avoid changing too many things at once.
* Have a clear commit history, possibly rebasing as needed.
* Make your commit messages clear and descriptive.

Note that Datadog requires all contributors to sign their commits. If you don't currenty sign your commits, follow [GitHub's documentation on how to set up your signing keys and start signing your commits](https://docs.github.com/en/authentication/managing-commit-signature-verification/signing-commits).
